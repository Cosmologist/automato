# Schedules Sync

Обновляет расписание сотрудников в `schedules.xlsx`.

## Структура xlsm

Один лист, две таблицы, разделённые пустой строкой.

### Таблица 1
- 2-row header + data rows
- Col A: № (с 1), Col B: ФИО, Col C+: часы/статусы по дням

### Таблица 2
- 1-row header (текстовый заголовок, не колонки) + data rows
- Col A: № (с 1), Col B: ФИО

## normalize_fio()
1. `\xa0` → пробел
2. `re.sub(r'\s+', ' ')` — схлопывание пробелов
3. `.strip()`
4. `ё→е`, `Ё→Е`

## fio_key()
`normalize_fio() + .lower()` — для case-insensitive сравнения ФИО.
Все сравнения ФИО (поиск в словарях/сетах, сортировка) используют `fio_key()`.

## clean_fio_cell()
1. `str(val)`, удалить `(ученик)`
2. Удалить префикс `\d+/\d+` (напр. `5/2`, `4/2`)
3. Удалить trailing `\d+`
4. `normalize_fio()`

## is_fio_cell()
После `clean_fio_cell()` — ровно 2 пробела (= 3 слова, кириллица).

## Режим `diff`

Full outer join списков ФИО из schedules.xlsx и employees.docx:
- совпавшие — default (без цвета)
- только в schedule — RED
- только в employees — GREEN

## Режим `clean`

Удаляет из schedules.xlsx строки сотрудников, у которых нет пары в employees.docx (RED из diff).
Использует `SheetMapping` — удаление через буфер с сохранением стилей/мержей.

## Парсинг xlsm
- Пустая строка — разделитель таблиц (первая встреченная)
- Table 1: данные rows[2:sep_idx]
- Table 2: данные rows[sep_idx+2:]
- Возвращает: `tables[].employees[].{row_num, fio_raw, fio}`

## Единый список (xlsm)
- `get_all_employees()` — объединяет обе таблицы, сортирует по `fio`
- Каждый employee содержит `table_index` (0 или 1)

## Парсинг docx
- Одна таблица, первая строка — заголовок
- Col 1 (index 1): ФИО (trailing `,` удаляется)
- Col 2 (index 2): метка
- Сортировка по `fio`
- `parse_employees()` → `[{fio_raw, fio, label}]`

## SheetMapping (удаление строк)

Поскольку openpyxl не сдвигает строки при удалении — делаем буфер:

1. `SheetMapping(ws)` — сканирует лист: значения, стили (font/fill/border/alignment), мержи, высоты/ширины
2. `mapping.remove_row(n)` — удаляет строку из буфера, сдвигает нижние строки вверх, корректирует мержи
3. `mapping.render(ws)` — очищает лист, пересоздаёт всё из буфера с полным применением стилей

Хранит только ячейки с value != None. Пустые форматированные строки не сохраняются.

### `bulk_insert_rows(after_rows)`
Единый проход: сдвигает все ячейки на количество вставок выше каждой строки.
Принимает список `after_rows` — позиций, после которых вставить пустые строки.
Возвращает `dict {after_row: final_position}`. Гарантирует, что gap-ы не затираются сдвигаемыми ячейками.

## Режим `collect`

Сканирует `departs/*.xlsx` и собирает данные сотрудников:

1. **Спиральный поиск ФИО** — clockwise от A1:
   - N=1: A1
   - N=2: B1, B2, A2
   - N=3: C1, C2, C3, B3, A3
   - N=4: D1, D2, D3, D4, C4, B4, A4 ...
2. **Очистка ячейки** через `clean_fio_cell()`, проверка `is_fio_cell()`
3. После нахождения — сбор строк вниз до пустой
4. Для каждого сотрудника: ФИО + данные на `working_days` колонок вправо

### Рендер в лист `schedule` (schedules.xlsx)
- Удаляется старый лист `schedule`, создаётся новый
- Col A: №, Col B: ФИО, Col C+: часы/статусы
- Строки без данных (только ФИО) пропускаются
- Если ФИО встречается в нескольких файлах — не перезаписывается, добавляется новой строкой

## Режим `merge`

Объединяет Clean + schedule + таблицу 2 первого листа в единый лист `Final`:

1. **Clean** (полные стили) — таблица 1 как основа
2. **schedule** (только значения) — данные для слияния
3. **Первый лист** (полные стили) — таблица 2 в конец

### Логика слияния таблицы 1
- За основание берётся таблица 1 из Clean
- Совпавшие по ФИО из schedule — перенос значений C+
- Без пары из schedule — вставка в алфавитном порядке, стили от ближайшего соседа, **красная** заливка A и B
- Без пары из Clean — **оранжевая** заливка всей строки

### Рендер
- Таблица 2 первого листа добавляется после разделителя
- Перенумерация колонки A для каждой таблицы
- Результат — лист `Final`
