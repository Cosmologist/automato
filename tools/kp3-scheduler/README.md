# kp3-scheduler
Обновляет расписание сотрудников

## Входные данные

### Основной файл расписания ./schedules.xlsx

#### Структура
Содержит график сотрудников на очередной месяц по дням.
Состоит из двух таблиц разделенных между собой пустой строкой.
Столбцы:
- Нумерация записей таблицы - 1 столбец
- ФИО - 1 столбец
- Данные графика на каждый день - от 28 до 31 столбцов (в зависимости от месяца на который составляется график)

#### Интерфейс
- Предоставляет список всех нормализованных ФИО из первой и второй таблицы в алфавитном порядке
- Предоставляет SheetMapping первой и второй таблицы
- Позволяет очистить значения графика в SheetMapping
- Позволяет проверять наличие ФИО в списке
- Позволяет установить значения графика по ФИО в SheetMapping
- Позволяет добавить новую строку с ФИО и графиком в SheetMapping, с сохранением алфавитного порядка, в первую или вторую таблицу
- Позволяет подсветить ячейку (бэкграунд) с ФИО цветом в SheetMapping
- возвращает количество дней графика (количество заполненных ячеек С2:AF2)


### Файл с актуальным список сотрудников ./employees.docx

#### Структура 
Содержит внутри себя таблицу с актуальным списком сотрудников
Столбцы таблицы:
- ФИО - 1 столбец
- Метка - 1 столбец

#### Интерфейс
- Предоставляет список всех нормализованных ФИО сотрудников в алфавитном порядке с меткой

### Файлы с графиками сотрудников по отделам ./departs/*.xlsx (для режима merge) и ./update/*.xlsx (для режима update)

#### Структура
Содержат список сотрудников отдела - каждый файл отличается немного от другого, 
но их объединяет то, что каждый из них содержит матрицу из одного столбца с ФИО + от 28 до 31 столбца с данными графика, справа от фио.

#### Алгоритм поиска данных
Для каждого файла используя спиральный поиск ищем первую ячейку с ФИО - это верхний левый угол матрицы с данными.
Выбираем все данные графика из ячеек справа от ФИО.
Выбираем данные по следующему сотруднику из строки ниже тем же образом пока не дойдем до конца данных (полностью пустой строки) - пропускаем строки где ФИО не проходит проверку на ФИО.

#### Алгоритм спирального поиска
Спиральный поиск первой ячейки удовлетворяющей условию - перебор ячеек по спирали (clockwise) начиная с A1 до первой ячейки:
- N=1: A1
- N=2: B1, B2, A2
- N=3: C1, C2, C3, B3, A3
- N=4: D1, D2, D3, D4, C4, B4, A4 ...

#### Интерфейс
- Предоставляет список всех нормализованных ФИО сотрудников в алфавитном порядке
- Предоставляет вместе с ФИО указанное значение графика (от 28 до 31)
- Данные из скрытых строк пропускаются
- Строки для которых все значения графика в указанном количестве равны "НН" (регистронезависимо)

### Файл с исправлениями по фио ./fio-alternatives.txt
Так как employees.xslx и employees.xlsx наши файлы, а depars/*.xlsx - сторонние и не контролируемые с нашей стороны,
то бывает разные написания ФИО.

#### Структура
- этот файл содержит строки вида "ФИО альтернативное написание | ФИО правильное написание"
- может содержать строку вида "ФИО |" - после разделителя ничего нет - значит указанную строку не считать ФИО при проверке "это ФИО?" 

## Компоненты

### Компонент SheetMapping
Абстракция над листом таблицы - наше внутреннее отображение данных листа.
Представляет матрицу NxM - где N - это актуальное количество строк, M - столбцов, 
Каждый элемент матрицы - отображение соответствующей ячейки листа вида:
- значение
- метаинформация - стили, заливка, цвет, шрифт, размер шрифта, контур ячейки и тп.

#### Методы 
Методы load() - загрузка листа таблицы.
Метод remove() - удалить соответствующие строки по условию
Метод insert() - позволяет вставить новую строку с данными или с данными и метаданными в нужную позицию, при этом оформление, высота строки и тп берется из ближайшей строки с данными
Методы render() - перерисовка внутреннего представления на лист таблицы (перед выполнением содержимое листа полностью очищается и сбрасыавется).

### Компонент ФИО

#### Нормализация ФИО
- поиск верного написания из ./fio-alternatives.txt
- удалить суб-строки "(ученик)"
- заменить "ё" на "е"
- удалить суб-строки формата "5\2" - где 5 и 2 могут быть любыми цифрами
- все пробельные и невидимые символы заменяются на пробел
- убираются все задвоения пробелов
- strip

#### Сравнение двух ФИО на равенство
- фио приводятся к lowercase после нормализации

### Это ФИО?
- Содержит ровно 2 пробела (= 3 слова, кириллица, без учета регистра) после нормализации 

## Шаги работы

### 1. Diff (режим по-умолчанию)
- Full outer join списков ФИО из schedules.xlsx, SheetMapping(schedules.xlsx) и списка из employees.docx
- Получаем список пар [ФИО schedules|null, ФИО employess|null, метка employees]
- Пары должны быть отсортированы по алфавиту по обоим ФИО сразу
- Вывод в консоль на проверку как таблицу вида: ФИО schedules, ФИО employees, метка employees
- Запрос пользователя на продолжение работы - этап Clean

### 2. Clean
- Для всех пар у которых ФИО employess=null - удаляем соответствующие строки SheetMapping(schedules.xlsx)
- Рендер SheetMapping(schedules.xlsx) в лист с именем Clean в schedules.xlsx
- Запрос пользователя на продолжение работы - этап Merge

### 3. Merge
Работает с файлами из ./departs/*.xlsx.

- Обнуляем все значения графика в SheetMapping(schedules.xlsx) - кроме заголовков и ФИО, и ячеек с значением "ПДН"
- делаем промежуточное сохранение в List Empty

- Проходим по всем ./departs/*.xlsx
- если SheetMapping(schedules.xlsx) ни первой ни второй таблицы не содержит такого ФИО
  - вставляем его в SheetMapping(schedules.xlsx) первой таблицы - посвечиваем ФИО зеленым
- если SheetMapping(schedules.xlsx) первой таблицы содержит такое ФИО - заменяем графиком из departs график в SheetMapping(schedules.xlsx)
- Собрать ФИО которые не найдены ни в одном ./departs/*.xlsx:
  - вывести предупреждение по ним
   - предложить в интерактивном режиме добавить в ./fio-alternatives.txt - если добавляем, то потом удаляем лист Clean, завершаем работу, выводим сообщение пользователю о необходимости перезапуска 
   - подсветить ячейку с фио красным в SheetMapping(schedules.xlsx)
- Запрос пользователя на продолжение работы
- Рендер SheetMapping(schedules.xlsx) в лист с именем Merge в schedules.xlsx
- Запрос пользователя на продолжение работы - этап Pretty

### 4. Update (режим запускается явно) 
Аналогичен режиму Merge, работает с файлами ./updates/*.xlsx.
Не участвует в режимах Diff и Merge.
Отличия от режима Merge:
- Работает только с первым листом - проверяет что только один лист в schedules.xlsx
- не обнуляет текущие значения графика в SheetMapping(schedules.xlsx)
- не обрабатывает ФИО которых нет ни в одном ./updates/*.xlsx
- Полностью очищает первый лист перед рендером
- Рендер SheetMapping(schedules.xlsx) в лист с именем Update в schedules.xlsx
- Запрос пользователя на продолжение работы - этап Pretty 

### 5. Pretty
- Пробегаем по всем данным графика SheetMapping(schedules.xlsx) первой таблицы после всех изменений
- если ячейка c значением "ОТ" делаем ее значение "ОТП"
- если ячейка c значением "В" очищаем значение в пустую строку
- Если ячейка пустая - делаем ее бэкграунд #B8CCE4
- Если ячейка с числом делаем ее бэкграунд white
- Если ячейка со значением "ОТП" - делаем ее бэкграунд #92D050
- Рендер SheetMapping(schedules.xlsx) в лист Pretty schedules.xlsx
- Сохраняемся
- Удаляем все листы кроме Pretty
- Переименовываем лист Pretty в Лист
- Сохраняемся

### 6. Пересчет нумерации (режим может запускаться отдельно)
Пересчет нумерации для двух таблиц на первом листе

Сообщаем об успешном завершении
